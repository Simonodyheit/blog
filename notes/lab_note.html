    # 服务器联网

    一个典型的案例是从互联网下载数据集前，你需要在服务器上登陆网络通账号，以让服务器接入互联网。在以往操作中，我们往往使用Teamviewer连接服务器，在图形界面的网站上进行连接，现在，可以完全舍去这种操作，直接在命令行进行连接：

    相关指南 (opens new window)

    curl --data "name=用户名&password=密码&cmd=set&type=出口&exp=时间" \
http://wlt.ustc.edu.cn/cgi-bin/ip

脚本说明：
  type =
     0 -- 教育网出口
     1 -- 电信出口
     2 -- 联通出口
     3 -- 电信出口2
     4 -- 联通出口2
     5 -- 电信出口3
     6 -- 联通出口3
     7 -- 教育网出口2
     8 -- 移动测试国际出口
  exp =
    0     -- 永久
    120   -- 动态
    3600  -- 1小时
    14400 -- 4小时
    39600 -- 11小时
    50400 -- 14小时

    提示：

    1. 服务器在校园网的内网中，所以服务器即使不连互联网，本地的电脑也能与之相连。但如果要在服务器上从互联网下载数据集，就要用以上指令。
    2. 当放假在家时，本地电脑脱离了校园网环境，服务器不连互联网是不可以和本地连接的。在家登录服务器时需要联系在校的实验室的同学协助在服务器连一下网络通。
    3. 服务器有时候连接超时，要考虑服务器是否连上了互联网，简单的判断方法就是通过ping www.baidu.com 测试是否能够ping通，如果长时间没有反应，说明没有联网。

    # conda虚拟环境

    以从Github上clone下来的项目为例，不同的项目对应的库/包（Pytorch、numpy）版本不同，而Linux又是支持多用户共同访问的，所以如若大家都是在默认环境里下载、安装对应的库/包等，容易产生冲突。所以对于每个人，甚至是每个项目而言，需要特别地为其创建一个虚拟conda环境，所有库/包都在该环境中下载、安装、使用，与主环境隔离开。

    关于conda虚拟环境的建立、进入、退出、删除（慎重！）可以参照网上资源。

    当你成功创建自己的虚拟环境并成功进入后，在终端的最左方会出现（yourCondaEnvName）的字样，这时使用命令conda env list时你会发现一个星号会出现在你命名的虚拟环境前，当成功做到这一步时，请继续阅读此节后面内容。

    有两点需要注意，一是没有经过同意，最好不要使用他人的虚拟环境，相关配置的改变将令人非常头疼。

    第二点是，在conda虚拟环境中你将会安装大量的库/包用于训练神经网络。而在安装Pytorch时，要去官方寻找与cuda适配的版本，下面将讲解具体如何操作。

    # tmux窗口

    很好，相信你已经成功进入了自己的虚拟环境中，并很熟练的在每次登录访问服务器时都会conda activate进去。

    现在假使一切环境依赖准备就绪，你的第一个网络也成功跑起来了，问题来了，它太耗时了。

    对于一个程序，如果其运行时间过长，比如从网上下载一个大型数据集、训练某神经网络，直接在当前虚拟环境下的普通终端里不是最佳选项。这是因为如果你退出连接服务器的客户端（比如VS Code, MobaXterm, XShell）时，正在运行的程序（在Linux上称为进程）会被中止，所以我们需要将其置于后台运行。而且我们还希望下次再进来的时候，能看到训练的进度如何（如果代码支持屏幕输出进度）。

    这就是tmux终端的作用。在当前普通终端输入tmux指令并回车就可进入一个名称为数字编号的tmux窗口，在这里面运行耗时较长的程序后，就可以直接将客户端关掉。下次再登陆服务器时，可以进入之前创建的tmux窗口查看。

    但其缺陷是不支持滚轮翻动大片输出，比如当你要在屏幕上打印显示网络模型结构时，在tmux窗口中打印就不是最佳选择，你只能看到部分输出，而翻不到上面的内容。在这种情况下，要在普通终端（也就是退出tmux窗口后）打印输出，对程序进行调试亦然。

    总结一下：想在后台执行的程序都可以放在tmux窗口执行，而且这里并不适合大片屏幕输出。

    常用tmux命令 
    
tmux ls # 查看此用户名（whoami命令可以查看）下有多少tmux窗口
tmux # 进入一个tmux窗口，其名称为数字编号
tmux a # 进入上次离开的tmux窗口
tmux new -s xxx # 创建一个名为xxx的窗口并进入
tmux detach # 离开当前窗口，回到主终端（普通终端），该tmux还保留，用tmux ls可以看到新建的tmux窗口
exit # 在tmux窗口里面退出回到主终端，此tmux终端被销毁
tmux kill-session -t xxx # 删除xxx的tmux，在tmux窗口外销毁
tmux attach-session -t xxx # 进入指定名称的tmux窗口
tmux at -t xxx # 精简版进入指定名称的tmux窗口

    # 指令自动化

    对于一些重复的批量操作，或者参数很多的命令，可以利用脚本文件执行。在Linux系统中，脚本文件以.sh结尾。

    比如一个训练指令 python trainer_vanilla.py --proj_name Vanilla --model_type Vanilla --data_type FashionMNIST --output_dir ./Results/Vanilla/ --gpu_devices 1 --data_dir ./dataset, 由于需要很多配置，造成指令较长，将其置入一个script.sh中，并按照需求改动里面的配置参数。

    ## this is a bash file named script.sh
python trainer_vanilla.py --proj_name Vanilla --model_type Vanilla --data_type FashionMNIST --output_dir ./Results/Vanilla/ --gpu_devices 1 --data_dir ./dataset

    适当使用\换行能让结构更清晰，注意\后面不能有空格，前面必须空一格。

    ## Better practice: this is a bash file named script.sh
python trainer_vanilla.py \
--proj_name Vanilla \
--model_type Vanilla \
--data_type FashionMNIST \
--output_dir ./Results/Vanilla/ \
--gpu_devices 1 \
--data_dir ./dataset

    有了bash文件后，在bash文件所在文件夹下使用指令sh script.sh即可运行。

    这里仅展示了bash文件解决一条长配置参数命令的问题，当然bash也能够依顺序执行一系列命令。更多高级用法如设置变量等，大家可以去自行学习。

    # 一、注意事项

    # 1. 内存空间

    由于home文件夹下内存吃紧，建议大家在/mnt/xxx (如/mnt/data3/, /mnt/data4/)下创建个人文件夹，并将个人资料、数据、代码等存在该文件夹下。如有需求，定期做好备份。

    # 2. 数据集

    • 下载前

      在下载公开数据集之前，应当确认服务器上是否已存在该数据集，如果存在，应当直接使用。

      • [ ] 整合所有数据集到同一文件夹中。
      • [x] /mnt/data3/Public/PytorchTemplate涉及数据集集中存放在/mnt/data4/Dataset/

    • 准备下载

      如果你已经确认要下载数据集，请确认当前位置（pwd输出）是在目标文件夹下。

      ⚠️注意：请不要在/home/及子文件夹下下载数据集。

      推荐在tmux窗口中下载。tmux窗口支持后台挂起，避免下载过程中因为与服务器断开连接造成下载中断/失败。

    • 下载后

      如无必要，请只使用而不要更改数据集（数据增强不在此范围内，此时数据集已被导入，原数据集不会受影响）。

    # 3. 删除

    删除风险很大，请慎用rm指令，尤其是如rm -r *.txt使用通配符的指令，这在有些情况下可能会删除意料之外的文件/文件夹，引起不必要的麻烦。

    在下载和解压数据集时，容易犯的错误就是直接将数据集下载/解压在home/ustc115/下，服务器出现卡顿情况。究其原因，是因为所有ustc115用户登陆成功后默认所在地址是home/ustc115，解压的结果直接输出到当前文件夹下，导致内存被挤占。

    现要求将该文件夹下所有的解压后的数据集全部删除，因为个数较多，无法一一手动删除，已知文件后缀“.nii.gz”

    为防止误删，推荐先用查找find指令，确定查找到的结果是自己的想要匹配的文件，再删除，主要步骤如下：

    • 观察每个文件的命名格式，发现前缀都是IXI， 后缀都是nii.gz，故使用匹配字符串“IXI*.nii.gz”，其中* 表示任意类型/个数字符。

      find home/ustc115/ -type -f -name "IXI*.nii.gz"

    • 观察输出结果，确定都已包含在内。

    • 使用指令find home/ustc115/ -type -f -name "IXI*.nii.gz" | xargs rm 快速删除

    本质上，这等同于rm IXI*.nii.gz，但先find后删除的方式更安全。

    当然，根据需要，这里更合适的应该是将数据集移动到mnt下，避免再一次解压以及产生的潜在风险，即

    mv IXI*.nii.gz /mnt/data3/dataset 将所有符合匹配条件的文件移动到目标文件夹mnt/data3/dataset/中。

    kill -9 xxx 在外部终止某进程，其中xxx为进程号，依据为top指令

    # 文件夹的信息（内容、结构、个数、权限）

    ls输出当前地址的所有文件/文件夹名称。

    ll显示更多信息。

    两者都可以查看当前文件夹下有哪些文件和文件夹。

    此外，还可以

    ls | wc -l 统计当前文件夹中的文件/文件夹个数。

    ls /path/to/the/dir/ | wc -l 统计/path/to/the/dir/中的文件/文件夹个数。

    du -h --max-depth=1 /path/to/the/dir/ 查看/path/to/the/dir一级目录下各文件/文件夹的大小，如果不指定就是当前文件夹）

    df -h 系统各个文件夹内存使用情况。

    tree 输出当前文件夹及子文件夹的结构。

    典型应用场景是在Github上clone某个仓库到服务器上后，使用tree指令查看该仓库的结构。

    # 当前文件夹与进入指定文件夹

    pwd输出当前所在地址。

    cd . 不做操作，因为.表示当前文件夹

    cd ..进入上一层文件夹

    cd xxx 前往目标文件夹，对于当前文件夹的子文件夹，直接提供文件夹名称即可；其他情况需要绝对地址。

    注意：涉及进入文件夹的操作经常和上键（历史指令），以及Tab键（自动补全）共同使用。

    比如当前文件夹是/aa/bb/cc/dd，要进入/aa/中，可以直接cd /aa/, 也可以cd ..-->回车-->上键-->回车-->上键-->回车。

    # 监控进程

    这里的进程主要指的是正在运行的程序。

    top 用于动态监控服务器各用户、对应进程，其会定时自动刷新。按c键可以显示详细的命令，退出按q键。由于其输出占显示器面积较大，一般退出后使用ctrl + L清屏。

    静态监控命令为ps aux，其不会自动刷新并显示动态的进程信息。

    典型使用场景是当某进程卡死，无法在内部删除时，通常使用top指令查找对应的PID号，并使用kill -9 PID号强制杀死进程。

    nvidia-smi 查看服务器上显卡GPU的使用情况。

    典型应用场景是查看服务器上是否还有显卡用于训练神经网络模型，卡的剩余内存为多少，通过实验确认可以支持多大的batch_size等等。

    对于静态指令如ps aux 和nvidia-smi, 需要手动多次执行该命令来看状态变化情况，而一种自动化方案是watch -n 5 nvidia-smi，每隔5秒刷新显示nvidia-smi结果，退出使用ctrl + C。

    # 寻找与匹配

    find . -maxdepth 1 -type f | wc -l 输出当前文件夹下的所有文件个数（上方ls不区分文件和文件夹）。

    find . -maxdepth 1 -type d | wc -l 输入当前文件夹下的所有文件夹个数

    find path/to/dir -type d -name "xxx" 在path/to/dir文件夹及子文件中寻找名为xxx的文件夹。

    grep 指令在指定文件/多个文件/文件夹中搜索匹配字符。https://phoenixnap.com/kb/grep-command-linux-unix-examples#ftoc-heading-5

    一种应用场景是在测试结果中查找某特定指标。
