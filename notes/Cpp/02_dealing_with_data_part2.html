    # Floating-point Numbers

    # What floating?

    A computer stores a number with fractional part in two parts. One part represents a value, while the other is scale. 34.1245 and 24124.5 are identical except for scale.
    The scaling factor servers to move the decimal point, hence the term floating-point.

    # Why floating?

    1. It enables to represent numbers with fractional parts.
    2. It can represent large numbers with which long fails.
    3. This holds true to very small values.

    # Notation

    1. . the period
    2. E or e
    2.52e+8
8.33E-4
7E5
-18.32e13
1.69e12

    # Types

    1. float
    2. double
    3. long double
      These types are described in terms of the number of significant figures(the meaningful digits in a number) they can represent and the minimum allowable range of exponents.

    # Floating-Point Constants

    By default, floating-point constants such as 8.24 and 2.4E8 are type double.

    Remember, floating-point constants are type double by default.

    Specify the type as follows:

    1.234f // a float constant
2.45E20F // a float constant
2.345324E28 // a double constant
2.2L // a long double constant
    Why specify?

    Once you define a variable in the C++ program, you need also specify the data type. So on which scenario should one use the suffix to specify the data type? (cout and a pure number with no assignment of it to a variable)

    # C++ Arithmetic Operators

    # Five basic arithmetic operators

    1. + ---> add
    2. - ---> subtract
    3. * ---> multiply
    4. / ---> divide and integer part if both operands are integers
    5. % ---> modulus

    # Order of Operation: Operator Precedence and Associativity

    1. Operator Precedence int a = 3 + 4 * 5; is int a = 3 + (4 * 5);
    lingo
    1. Use the parenthesis to enforce the priorities.
    2. *, /, % have equal/the same precedence.
    3. + and - share a lower precedence.
    1. Associativity int b = 120 / 4 * 5; is int b = (120 / 4) * 5;

    A case study:

    int dues = 20 * 5 + 24 * 6;

    Order precedence tells us that 20 * 5 and 24 * 6 take place first against the +. However, the associativity does not say 20 * 5 takes place first since it works only when two operators share the same operands. The two * operators do not share a common operand, so the rule fails to apply.

    In fact, C++ leaves it to the implementation to decide which order works best on a system. For this example, either order gives the same result, but there are circumstances in which the order can make a difference.

    # Type Conversion

    FACTS

    1. There are 11 integer types and 3 floating-point types in C++.
    2. C++ makes type conversions automatically.
      1. C++ converts values when you assign a value of one arithmetic type to a variable of another arithmetic type.
      2. C++ converts values when you combine mixed types in expressions.
      3. C++ converts vaules when you pass arguments to functions.

    # Conversion on Initialization and Assignment

    Potential Problems Posed by Conversion

    1. Bigger floating-point type to smaller floating-type, such as double to float ----> Loss of precision; value might be out of range for target type, in which case result is undefined.
    2. Floating-point type to integer type ----> Loss of fractional part; Overflow issue as in the above.
    3. Bigger integer type to smaller integer type, such as long to short ----> Overflow issue; loss of bytes.
    // init.cpp -- type changes on initialization
#include <iostream>
int main(){
    using namespace std;
    cout.setf(ios_base::fixed, ios_base::floatfield);
    float tree = 3; // int converted to float
    int guess(3.9832); // double (default) converted to int
    int debt = 7.2E12; // result not defined
    cout << "tree = " << tree << endl;
    cout << "guess = " << guess << endl;
    cout << "debt = " << debt << endl; // undefined result varied from complier to compiler
    return 0;
}

    # Conversions in Expressions

    Integral promotion:

    1. C++ converts bool, char, unsigned char, and short vaules to int.
      short chickens = 20; // line1
short ducks = 35;  // line2
short fowl = chickens + ducks; // line3
    2. unsigned short is converted to int if short is smaller than int, otherwise to unsigned int.
    3. wchar_t is promoted to the first of the following types that is wide enough to accommodate its range: int, unsigned int, long, unsigned long.

    Rules

    1. If either operand is type long double, the other operand is converted to long double.
    2. Otherwise, if either operand is double, the other is converted to double.
    3. Otherwise, if either operand is float, the other to float.
    4. Otherwise, the operands are integer types (11 types!) and the integral promotions are made.
    5. In that case, if both operands are signed or if both are unsigned, and one is lower rank than the other, it is converted to the higher rank.
    6. Otherwise, one operand is signed and one is unsigned. If the unsinged operand is of higher rank, the latter is converted to the type of the unsigned operand.
    7. Otherwise, if the signed type can represent all values of the unsigned type, the unsigned operand is converted to the type of the signed type.
    8. Otherwise, both operands are converted to the unsigned version of the signed type.
