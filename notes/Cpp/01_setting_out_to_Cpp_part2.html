    Setting out to C++ Part II

    1/18/2022 C++basic

    Pearl quote

    A precise, recoginizable personal naming convention is a hallmark of good software engineering, and it will aid you throughout your programming career.

    # 1. C++ Statemnets

    A C+ program is a collection of functions, and each function is a collection of statements. Statements can be classified into 2 catergories:

    1. declaration statement
    2. assignment statement
    carrots.cpp 
    // carrots.cpp -- food processing program
// uses and displays a variable
#include <iostream>

int main()
{
    using namespace std;
    int carrots;
    carrots = 25;
    cout << "I have ";
    cout << carrots;
    cout << " carrots.";
    cout << endl;
    carrots = carrots - 1;
    cout << "Crunch, crunch. Now I have " << carrots << " carrots." << endl;
    return 0;
}

    # 1.1 Declaration Statements and Variables

    Use a declaration statement to indicate the type of storage and to provide a label for the location as in：

    int carrots; // "carrots' is a variable because one can change its value; it is also a label.

    Precisely, declarations can be further into defining declaration(int carrots;) and reference declaration (already defined).

    # 1.2 Assignment Statements

    An assignment statement assigns a value to a storage location. = is the assignment operator.

    You can use the assignment operator serially, as:

    int steinway;
int baldwin;
int yamaha;
yamaha = baldwin = steinway = 88; // working from right to left

    # 1.3 A New Trick for cout

    cout works with both strings and integers.

    Instead of inserting a string to the output flow as in

    cout << "Hello world!";

    one can also insert a varible:

    cout << carrots; // that is to say, one can also insert a integer, which is the data type of carrots
cout << "carrots"; // a totally different statement with the above one

    # 2. cin and cout: A Touch of Class

    • cin and cout are objects, respetively constructed by class istream and class ostream, as defined in the header file iostream(.h).
    • << is a way of calling the inner method.
    • FYI, there is a hidden conversion of data type in cin and cout. They are smart.

    A class bears the relationship to an object that a type does to a variable.
    The class describes all the properties of a data type, including actions that can be performed with it, and an object is an entity created according to that description.

    # 3. Functions Basics

    # 3.1 With/Without Return Value

    double x; // never forget to declare a variable before using it
x = sqrt(6.25);

    x = sqrt(6.25); is a statement or an expression.
    sqrt() is a function, also a called function.
    sqrt(6.25) is a function call.
    main() is the calling function.(Not explicitly here)

    In short

    The argument/parameter 6.25 is passed to the function, which returns a value back and get sented to a variable x by the assgiment operator =.

    More advanced usage of functions:

    double side = sqrt(area);

double pow(double); // prototype
answer = pow(5.0, 8.0);

int rand(void); // prototype
myGuess = rand();

void bucks(double); // prototype
bucks(1234.56)

    # 3.2 Function Prototype

    • A function prototype does for functions what a variable declaration does for variables.
    double sqrt(double); // a function prototype
    • When you use a function in a program, you must also provide the prototype in either of two ways:
      1. Explicit way. An explicit function prototype statement in the source code.
      2. Implicit way. Include the cmath heder file (in this case), which has the prototype within.

    The semicolon here is of quite importance, since it indicates that this statement is a function prototype, not a function header preceding a function body.

    # 3.3 User-Defined Functions

    • mian() is already an user-defined function. And it is a must to define.
    • Like C, and unlike Pascal, C++ does not allow you to embed one function definition inside another.
    • return 0;: the 0 is an exit value.
      The normal convention is that an exit value of zero means the program ran successfully, whereas a nonzero value means there was a problem.
    • Incidentally, main is not a keyword because it’s not part of the language. Instead, it is the name of a required function. You can use main as a variable name. (That can cause a problem in circumstances too esoteric to describe here, and because it is confusing in any case, you'd best not.)
    • Limiting std namespace access to just a function or not, this is a question. The current prevalent philosophy is that it's preferable to be more discriminating. In short, there are four ways:
      1. Above the function header
      2. In a function body
      3. Specify a particular element in a function body using std::cin;
      4. Discarding using namespace but always preceeding with std:: as in std::cout << "a string" << std:endl;
    A vanilla user-defined function without return 
    // myfunc.cpp -- defining your own function
#include <iostream>
void simon(int); // function prototype

int main()
{
    using namespace std;
    simon(3);
    cout << "Pick an integer: ";
    int count;
    cin >> count;
    simon(count);
    cout << "Done!" << endl;
    return 0;
}

void simon(int n) // simon()'s header file
{
    using namespace std;
    cout << "Simon says touch your toes " << n << " times." << endl;
}
    A vanilla user-defined function with return 
    // myfunc.cpp -- defining your own function
#include <iostream>
int simon(int); // function prototype

int main()
{
    using namespace std;
    cout << "Pick an integer: ";
    int count;
    cin >> count;
    int num = simon(count);
    cout << "Done! " << num << endl;
    return 0;
}

int simon(int n) // simon()'s header file
{
    using namespace std;
    return 3 * n;
}
