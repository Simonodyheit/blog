    # 1. Identify data

    But identify what?

    1. What kind of type is the data? ---> data type
    2. What is in the data? ---> value

    In whatever way, one must specify the first two information in advance in a program, but for the 3rd, it's a choice. & and pointer can be involved.

    # 3. Integer Types

    The various C++ integer types differ in the amount of memory they use to hold an integer. The more memory a value uses, the wider it is.

    1. char
    2. short
    3. int
    4. long
    5. long long And each comes in both signed and unsigned versions.
      Notes:
    • A short integer is at least 16 bits wide.
    • An int integer is at laest as big as short.
    • A long integer is at least 32 bits wide and at least as big as int.
    • A long long integer is at least 64 bits wide and at least as big as long.

    However, C++ defines byte differently. The C++ byte consists of at least enough adjacent bits to accommodate the basic character set for the implementation. That is, the number of possible values must equal or exceed the number of distinct characters. In the United States, the basic character sets are usually the ASCII and EBCDIC sets, each of which can be accommodated by 8 bits, so the C++ byte is typically 8 bits on systems using those character sets. However, international programming can require much larger character sets, such as Unicode, so some implementations may use a 16-bit byte or even a 32-bit byte. Some use the term octet to denote an 8-bit byte.

    To tell how much is which.

    // limits.cpp -- some integer limit
#include <iostream>
#include <climits>
int main(){
    using namespace std;
    int n_int = INT_MAX;
    short n_short = SHRT_MAX;
    long n_long = LONG_MAX;
    long long n_llong = LLONG_MAX;
    cout << "int is " << sizeof(n_int) << " bytes." << endl;
    cout << "short is " << sizeof(n_short) << " bytes." << endl;
    cout << "long is " << sizeof(n_long) << " bytes." << endl;
    cout << "long long is " << sizeof(n_llong) << " bytes." << endl;
    return 0;
}

    # 5. How C++ Decides What Type a Constant Is

    C++ stores integer constants as type int unless there is a reason to do otherwise. Two such reasons are if you use a special suffix to indicate a particular type or if a value is too large to be an int.

    More details of the suffixes:

    • l or L suffix ------> long; also there is likewise ll/LL
    • u or U suffix ------> unsigned int
    • ul or uL suffix -------> unsigned long; also there is likewise ull/uLL

    # 6. char

    char is also an integer type.

    #include <iostream>
using namespace std;
int main() {
    char ch;
    cout << "Enter a character: " << endl;
    cin >> ch;
	int nch = ch;
	cout << ch << endl;
	cout << nch << endl;
    return 0;
}

    In the above program, what is stored in ch and nch is exactly the same, it is the smart object cout determines which kind to print out based on the data type.

    # 6.2 signed char and unsigned char

    Unlike int, char is not signed by default, nor is it unsigned by default.

    char fodo; // maybe signed, maube unsigned
unsigned char bar; // unsigned
signed char snark; // signed

    To hold a standard ASCII character, it doesn't really matter whether char is signed or unsigned.

    But why?

    signed char type typically represents the range 0 to 255, and signed char type typically represents the range -128 to 127. Since there are only 127 representations, it does not matter. And also, the memory efficiency is not an issue; they are the same.

    # 6.3 wchar_t, char16_t, and char32_t

    1. wchar_t Ever wonder why char a = "Simon" does not work, that's because the memory space allocated to the data type char is far away enough for a string; it's for a character marked by single quote marks.

    The usual 8-bit char can represent the basic character set, and another type, called wchar_t, can represent the largest extended character set with sufficient space used on the system.

    wchar_t bob = L'P'; // wchar_t to declare the data type, L indicates the 'wide' property
wcout << L"tall\n"; // print out a wide-character string
    1. char16_t, and char32_t Because wchar_t's sign and size can vary across different implementations, more definite size and signedness data type for character is required.
    char16_t ch1 = u'q'; // 'u' for char16_t character and string
char32_t ch2 = U'\U0000222B';

    # 7. The const Qualifier

    A symbolic name can suggest what the constant represents:

    #define Month 12; // C style
const int Months = 12; // C++ style
// ! never do the following, this reports an error and think about why?
const int toes;
toes = 10;

    Once and for all

    The compiler does not let you subsequently change the value Months. If you try to, for example, g++ gives an error message that the program used an assignment of a read-only variable. (What is funny is that it's not variable agian, though being a variable.) The keyword const is termed a qualifier because it qualifies the meaning of a declaration.

    Why const

    Two reasons:

    1. Data type gets specified explicitly.
    2. C++'s scoping rules can limit the definition to a particular functions or files.
    3. const can be with more elaborate types, such as arrays and structures.
