    Setting out to C++ Part I

    1/17/2022 C++basic

    # 1. C++ Initiation

    A C++ vanilla program 
    // myfirst.cpp -- display a message
#include <iostream>             // a preprecessor directive 
int main()      // function header
{       // start of the function body
    using namespace std;        // make definition visible
    cout << "Come up and C++ me some time"; //message
    cout << endl;       // start a new line
    cout << "You won't regret it!" << endl;     // more output
    return 0; // terminate the main()
}       // end of function body

    # 1.1 Header Filenames iostream

    • No extension as the h in C ---> enter namespace.
    • Header file names from C to C++ start with c, like cmath.

    # 1.2 Function Header int main()

    • main() is called by startup code, so it's also a function.
    • int means main() returns an integer value to the calling function (in startup code)
    • () means that the function takes no information/argument.
    • Same as int main(void), pero this is not the case in C, which is TBD when nothing is given in the ().
    • return 0; is omittable only for main() function.
    • The compelling main the function name

      If you don't have main(), you don't have a complete standalone program.

    # 1.3 Namespaces

    If you use iostream instead of iostream.h, you should use using namespace std; the namespace directive to make the definition iostream available to your program.

    In this spirit, the classes, functions, and variables that are a standard component of C++ compilers are now placed in a namespace called std. This takes place in the h-free header files. This means, for example, that the cout variable (this should be named object) used for output and defined in iostream is really called std::cout and that endl is really std::endl. Thus, you can omit the using directive and, instead, code in the following style:

    std::cout << "Come up and C++ me some time.";
std::cout << std::endl;

    That's to say, overall have we three-fold approaches:

    1. iostream.h and cout
    2. iostream and std::cout
    3. iostream and cout under the namespace std (invoked by the statement using namespace std;(Q: where should one put the statement?)) In this spirit, one can even use using to make particular names available.
    using std::cout;
using std::cin;
using std::endl;
// the above three are euqal to one single statement below
using namespace std;

    # 1.4 C++ Output

    cout << "Come up and C++ me some time";
    • The "xxx" is a chracter string.
    • The statement inserts the string into the output stream.
    • The insertion operation << indicates that the string is sent to cout. It represents the information flow. And it's also a trigger to call the inner method of object's corresponding class if necessary.
    • cout is a predefined object managing displays, predefined in iostream.
    • endl is a manipulator, defined in iostream head file and is also part of the std namespace.
    cout << "You won't regret it!" << endl;
// same as 
cout << "You won't regret it!\n"

    # 1.5 Code Style

    1. One statement one line
    2. An opening brace and a closing brace for a function, each of which is on its own line
    3. Statements in a function indented from the braces
    4. No whitespace around the parentheses associated with a function name

    The fourth helps to differentiate functions from some built-in C++ structures, such as loops, that also use parentheses.
